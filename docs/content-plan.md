# Content Plan and Research Notes

## Goal
Build a complete, consistent, B2B-oriented content set for all public pages in TR/EN.

## Research Inputs
- Live site audit (existing content quality, empty pages, mismatched legacy blocks)
- Web references for structured standards and messaging direction:
  - EU Single-Use Plastics Directive context (plastic alternatives trend):
    https://eur-lex.europa.eu/eli/dir/2019/904/oj
  - Schema.org structured data reference:
    https://schema.org
  - llms.txt format guidance:
    https://llmstxt.org/
  - Google tag / conversion event documentation:
    https://support.google.com/tagmanager
    https://developers.facebook.com/docs/meta-pixel/

## Output Structure Applied
- Home: value proposition, trust metrics, service highlights, featured products, blog highlights.
- About/KVKK/Cookie/Privacy: long-form corporate/legal text with clear side info blocks.
- Services: service-by-service cards with bullets + process flow.
- Products: portfolio overview with MOQ/feature chips + detailed product page use-cases.
- Gallery: visual catalog-like product cards and captions.
- References: sector-based reference segmentation (without fake client claims).
- FAQ: expanded operational FAQ set (MOQ, lead time, sample, sectors, print).
- Blog: practical B2B articles for procurement/operations teams.
- Contact/Quote: clearer business context text + validated forms + UTM capture.

## Editorial Rules Applied
- No unverifiable client names.
- No copied paragraphs from external sites.
- TR/EN parity for key pages.
- Ads compatibility preserved (lead and intent events).
- llms.txt kept machine-readable and up-to-date with DB content.
